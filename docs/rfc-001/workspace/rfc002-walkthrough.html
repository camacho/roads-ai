RFC 002 · five principles8 / 16

The rules the selection lives by

Pick candidates at execution time, not now
Any shortlist written today goes stale — our own Ollama claim was already stale at first check (E18). A frozen "candidate snapshot" records who looked, where, and what was in/out.
Measure the teaching task
Leaderboard coding scores don't tell us whether a model can diagnose a beginner's bug or hold back a solution.
Evaluate what actually ships
Compression (E31), the chat template (E32), and the serving engine (E33) can each change behavior — so test the exact deployed form.
Smallest adequate choice, not smallest model
Quality gates entry; speed, capacity, and upkeep pick among the survivors.
Record evidence, not confidence
Fluent output or a nice model card proves nothing. Reproducible manifests and results do.
RFC 002 · eligibility9 / 16

What it takes to even enter the race

  • License: free for Roads to download, run, and self-host for <50 engineers. Read the actual terms — "open weights" on a label proves nothing.
  • Runs on the dev Mac — a pinned artifact under vLLM-Metal.
  • Feasible for the production server — a pinned representation exists for the Ubuntu/GPU target. The proven run happens later, at selection.
  • Complete artifact — weights plus tokenizer, config, chat template, helper files. Weights alone are not a candidate.
  • Speaks our API — supports the requests Trestle actually makes.
  • Handles our request sizes — measured, not assumed.

RFC 002 · parity12 / 16

Mac and server will never match word-for-word — by design we don't ask them to

Different hardware does floating-point math in different orders, so the same question can produce slightly different text (E33, the serving engine's own documentation).

So parity compares what the answer accomplished: was the diagnosis right, did it pick the same teaching mode, did it reset when it should? If those match, the backends agree where it matters. Where they differ materially, a human reviews — and the production result always wins.

RFC 002 · reproducibility13 / 16

Everything needed to rebuild the decision

For every finalist, the manifest records the exact, immutable versions of:

base model + license Mac artifact + hash production artifact + hash dev runtime server image digest tokenizer + template + settings prompt / skill / extension / pi revisions surface.json revision raw results + parity diff

Change any of these later — model revision, quantization, template, server image, prompt — and the affected gates rerun. Mutable names like latest are never records.
